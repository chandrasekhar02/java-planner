Java Collections Framework - In-Depth Learning Guide
==================================================

This document provides comprehensive explanations of Java Collections, focusing on List, Set, Map implementations, and proper iteration techniques.

1. COLLECTIONS OVERVIEW
=======================

The Java Collections Framework provides a unified architecture for storing and manipulating groups of objects. It consists of:
- Interfaces: Collection, List, Set, Map, Queue, etc.
- Implementations: ArrayList, LinkedList, HashSet, TreeSet, HashMap, etc.
- Algorithms: Sorting, searching, shuffling, etc.

Key benefits:
- Reduces programming effort
- Increases performance
- Provides interoperability between unrelated APIs
- Reduces effort to learn and use new APIs
- Increases software reuse

2. LIST COLLECTIONS
===================

List is an ordered collection that allows duplicate elements. Main implementations: ArrayList and LinkedList.

2.1 ArrayList
-------------
- Backed by a dynamic array
- Fast random access (O(1) for get/set operations)
- Slow insertions/deletions in the middle (O(n) - requires shifting elements)
- Good for: Frequent read operations, rare modifications
- Memory efficient for storing elements

Time Complexity:
- get(index): O(1)
- add(element): O(1) amortized (occasional array resizing)
- add(index, element): O(n)
- remove(index): O(n)
- contains(element): O(n)

2.2 LinkedList
--------------
- Doubly-linked list implementation
- Fast insertions/deletions anywhere in the list (O(1) if you have the position)
- Slow random access (O(n) - need to traverse from head/tail)
- Good for: Frequent insertions/deletions, queue/stack operations
- Extra memory for links between nodes

Time Complexity:
- get(index): O(n)
- add(element): O(1) at end
- add(index, element): O(n)
- remove(index): O(n)
- contains(element): O(n)

When to use ArrayList vs LinkedList:
- Use ArrayList when you need fast random access and few insertions/deletions
- Use LinkedList when you have many insertions/deletions, especially at the beginning/middle
- For small lists (< 100 elements), performance difference is negligible

3. SET COLLECTIONS
==================

Set is a collection that contains no duplicate elements. Main implementations: HashSet, TreeSet, LinkedHashSet.

3.1 HashSet
-----------
- Backed by HashMap (uses hash table)
- No ordering guarantee
- Fast operations (O(1) average case)
- Allows null element
- Best for: General-purpose set operations, when order doesn't matter

3.2 TreeSet
-----------
- Backed by TreeMap (red-black tree)
- Elements are sorted according to natural ordering or custom Comparator
- Slower operations (O(log n))
- No null elements allowed
- Best for: Ordered set operations, range queries

3.3 LinkedHashSet
-----------------
- Backed by LinkedHashMap
- Maintains insertion order
- Slightly slower than HashSet due to linked list overhead
- Best for: When you need predictable iteration order

4. MAP COLLECTIONS
==================

Map stores key-value pairs. Main implementations: HashMap, TreeMap, LinkedHashMap.

4.1 HashMap
-----------
- Backed by hash table
- No ordering guarantee
- Fast operations (O(1) average case)
- Allows one null key, multiple null values
- Best for: General-purpose map operations, when order doesn't matter

4.2 TreeMap
-----------
- Backed by red-black tree
- Keys are sorted according to natural ordering or custom Comparator
- Slower operations (O(log n))
- No null keys allowed
- Best for: Ordered map operations, range queries by key

4.3 LinkedHashMap
-----------------
- Backed by hash table with linked list
- Maintains insertion order
- Slightly slower than HashMap due to linked list overhead
- Allows one null key, multiple null values
- Best for: When you need predictable iteration order

Differences Summary:
- HashMap/HashSet: Fast, unordered
- TreeMap/TreeSet: Ordered, slower
- LinkedHashMap/LinkedHashSet: Ordered (insertion), medium performance

5. ITERATION TECHNIQUES
=======================

5.1 For-each Loop (Enhanced for Loop)
-------------------------------------
- Simplest way to iterate
- Works with any Collection that implements Iterable
- Cannot modify collection during iteration (ConcurrentModificationException)
- Cannot access iterator for removal

Example:
for (String item : collection) {
    System.out.println(item);
}

5.2 Iterator
------------
- More flexible than for-each
- Allows safe removal during iteration using iterator.remove()
- Can iterate in both directions (ListIterator for lists)
- Can check for next element without consuming it

Example:
Iterator<String> iterator = collection.iterator();
while (iterator.hasNext()) {
    String item = iterator.next();
    if (shouldRemove(item)) {
        iterator.remove(); // Safe removal
    }
}

5.3 ListIterator
----------------
- Extended Iterator for List collections
- Allows bidirectional traversal
- Can add, set, and remove elements during iteration
- Can get current position

Example:
ListIterator<String> listIterator = list.listIterator();
while (listIterator.hasNext()) {
    String item = listIterator.next();
    if (condition) {
        listIterator.set("new value");
    }
}

6. OTHER COLLECTIONS
===================

6.1 Queue
---------
- FIFO (First In, First Out) data structure
- Common implementations: LinkedList, PriorityQueue
- Main operations: offer() (add), poll() (remove), peek() (examine)

6.2 Stack
---------
- LIFO (Last In, First Out) data structure
- Extends Vector (synchronized, not recommended for new code)
- Main operations: push(), pop(), peek()

6.3 Deque
---------
- Double-ended queue
- Can add/remove from both ends
- ArrayDeque is the recommended implementation
- Can be used as both queue and stack

7. BEST PRACTICES
=================

1. Choose the right collection based on your needs:
   - Need ordering? Use TreeSet/TreeMap
   - Need insertion order? Use LinkedHashSet/LinkedHashMap
   - Need fast lookup? Use HashSet/HashMap
   - Need fast random access? Use ArrayList
   - Need fast insertions/deletions? Use LinkedList

2. Prefer interfaces over implementations:
   - Use List instead of ArrayList
   - Use Set instead of HashSet
   - Use Map instead of HashMap

3. Use generics to ensure type safety

4. When iterating:
   - Use for-each for simple read-only iteration
   - Use Iterator when you need to remove elements during iteration
   - Use ListIterator for bidirectional traversal of lists

5. Consider thread safety:
   - Collections.synchronizedList() for synchronized collections
   - Concurrent collections for high-concurrency scenarios

8. COMMON PITFALLS
==================

1. ConcurrentModificationException:
   - Occurs when modifying collection during iteration
   - Solution: Use iterator.remove() or create a copy

2. Choosing wrong implementation:
   - Using ArrayList for frequent middle insertions
   - Using TreeMap when ordering isn't needed

3. Not using generics:
   - Leads to ClassCastException at runtime

4. Ignoring performance characteristics:
   - HashMap for small datasets where TreeMap ordering is needed

9. PERFORMANCE CONSIDERATIONS
=============================

- For small collections (< 10 elements): Implementation choice doesn't matter much
- For large collections: Choose based on access patterns
- Hash-based collections: O(1) average, O(n) worst case (hash collisions)
- Tree-based collections: O(log n) for all operations
- Array-based: Fast access, slow modifications
- Linked: Fast modifications, slow access

10. JAVA 8+ FEATURES
===================

- Streams API for functional-style operations
- forEach() method on collections
- Collectors for creating collections from streams
- Parallel streams for concurrent processing

Example:
collection.stream()
    .filter(item -> item.length() > 5)
    .sorted()
    .forEach(System.out::println);
